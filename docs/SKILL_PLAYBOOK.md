# Skill Playbook For Future Sessions

Last updated: 2026-04-20

These are the most important installed skills to use on this project and when to use them.

## Always use first

- `Superpowers`
  - default workflow discipline, context compression, and execution hygiene
- `karpathy-guidelines`
  - use for every coding task
  - think before coding
  - keep changes surgical
  - define verifiable success criteria

## Before changing design or UX

- `frontend-design`
  - primary design driver for public pages
  - use especially on:
    - homepage
    - data visualizations
    - evidence patterns
    - fight back
    - share your story

## Before implementation

- `project-planner`
  - use to define the sprint scope and update `PLAN.md`
- `executing-plans`
  - use while carrying the plan through page by page
- `brainstorming`
  - use before creative work or major UX changes

## While coding

- `test-driven-development`
  - add or update regression coverage for every bugfix or major UI/data behavior change
- `requesting-code-review`
  - use before claiming a sprint is done
- `verification-before-completion`
  - always run verification commands before saying a fix shipped

## For browser and live-product work

- `webapp-testing`
  - use when validating real flows and visual regressions

## For audits

- `audit-context-building`
  - use before deep code/security review
- `insecure-defaults`
- `supply-chain-risk-auditor`
- `differential-review`

## How to apply them here

The product is at its best when skills are used in this order:

1. `brainstorming`
2. `project-planner`
3. `frontend-design`
4. `karpathy-guidelines`
5. `test-driven-development`
6. `executing-plans`
7. `requesting-code-review`
8. `verification-before-completion`

## Warning

Do not claim a skill was “used” if the page still looks unchanged. On this repo, the user has already called out superficial or non-visible design work. Use the skills to create visible, testable product improvements.
